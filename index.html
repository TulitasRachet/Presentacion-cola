Descripción del TAD Cola

Estructuras de Datos: Conceptos fundamentales, implementaciones estáticas (Circulares), dinámicas y sus aplicaciones en el Sistema Operativo.

Concepto: Listas FIFO (First In First Out)

Una COLA es un tipo especial de lista lineal, un concepto que abunda en la vida cotidiana (bancos, supermercado).

  • Dos extremos: Funciona exclusivamente a través de dos puntos de acceso: FRENTE y FINAL.
  • Llegadas (Encolar): Todo elemento nuevo se ubica siempre al FINAL.
  • Salidas (Desencolar): Los elementos siempre se atienden y eliminan desde el FRENTE.
  • Política FIFO: El primero en llegar será el primero en salir.

/* FIFO en acción */

Especificación del TAD Cola

  • Inicializar (Initialize): Recibe una cola y la inicializa (prepara apuntadores).
  • Encolar (Queue): Agrega un elemento nuevo exclusivamente al final. El final aumenta.
  • Desencolar (Dequeue): Remueve el elemento del frente de la cola y lo retorna.
  • Es vacía (Empty): Devuelve verdadero si no contiene elementos.
  • Frente / Final: Retornan una lectura de los elementos ubicados en los extremos sin eliminarlos.
  • Eliminar (Destroy): Libera completamente su espacio en memoria.

Control de Errores: Estados Críticos

Subdesbordamiento (Underflow)

Al remover el último elemento, la cola queda vacía. Una vez vacía, no se pueden sacar elementos.

Cualquier intento de desencolar o acceder a elementos de una Cola Vacía causa un Subdesbordamiento (error fatal en ejecución).

Desbordamiento (Overflow)

Aplica principalmente en implementaciones estáticas. Ocurre cuando el hardware o arreglo llega a su límite de capacidad.

Cualquier intento de introducir elementos a una cola que ha llegado a su MÁXIMO_TAMAÑO (Cola Llena) causa un Desbordamiento.

Implementación Estática (Arreglos)

Se utiliza un arreglo estático de tamaño definido MAX.

  • Frente: De dónde se extraen los elementos.
  • Final: Dónde se introducen.
  • El Problema: Al extraer del frente, la posición 0 queda vacía. El final sigue avanzando y marca "Cola Llena" falsamente.
  • Recorrer todos los elementos a la izquierda cada vez que sacamos uno, consume demasiado procesamiento.

La Solución: Cola Circular

El objetivo es aprovechar al máximo el espacio del arreglo sin desplazar los datos.

  • Idea Principal: Insertar elementos nuevos en las localidades previamente desocupadas al inicio.
  • Reinicio del Apuntador: Al llegar al final lógico, si hay espacio libre adelante, el apuntador `Final` se reinicia a 0.
  • Se implementa usando el operador de módulo (%), formando lógicamente un "anillo" infinito.

Implementación Dinámica

Representación mediante apuntadores y Nodos para eliminar por completo la restricción de tamaño máximo de los arreglos estáticos.

Estructura Interna Dinámica

1. Estructura del Nodo (Enlace)

Cada bloque (nodo) se reserva sobre la marcha en la RAM y apunta al siguiente.

typedef struct Nodo { int dato; struct Nodo* siguiente; } Nodo;

2. Cabecera (Cola)

Mantiene el control absoluto de dónde empieza (Frente) y dónde termina (Final).

typedef struct Cola { Nodo *frente; Nodo *final; } Cola;

Operaciones Dinámicas

Queue (Encolar)

void Queue(Cola* q, int val) { Nodo* temp = crearNodo(val); if (q->final == NULL) { q->frente = q->final = temp; return; } q->final->siguiente = temp; q->final = temp; }

Dequeue (Desencolar)

int Dequeue(Cola* q) { if (q->frente == NULL) return -1; Nodo* temp = q->frente; int dato = temp->dato; q->frente = q->frente->siguiente; if (q->frente == NULL) q->final = NULL; free(temp); // Libera RAM return dato; }

Aplicaciones en el Sistema Operativo

Las colas se utilizan fundamentalmente cuando varios procesos deben esperar por un recurso o servicio de hardware.

  • Recursos Compartidos (Spooling): Colas de impresión. Los documentos esperan ordenadamente su turno para salir a la impresora física.
  • Ready Queue (CPU): El administrador de procesos gestiona el procesador. Los programas abiertos se forman en la cola de procesos listos para recibir su "turno" de ejecución (multitasking).
